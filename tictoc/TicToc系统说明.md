## 系统概述

**项目名称：** TicTocNet
**项目类型：** OMNeT++ 入门示例
**功能描述：** 两个网络节点（tic 和 toc）通过来回交换消息进行点对点通信仿真
**仿真时长：** 默认 1000 秒（或达到消息数量限制）
**源码位置：** `/home/sfm/文档/omnet-project-1/tictoc/`

## 模块功能详解

### 1. Txc 模块（`src/Txc.ned` + `src/Txc.cc`）

Txc 是整个系统的核心模块，采用 **简单模块（Simple Module）** 架构。

#### 参数定义

| 参数名 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `limit` | int | 10 | 最大接收消息数量，达到后停止接收 |

#### 端口定义

| 端口名 | 类型 | 方向 | 说明 |
|--------|------|------|------|
| `in` | input | 输入 | 接收来自其他模块的消息 |
| `out` | output | 输出 | 发送消息到其他模块 |

#### 核心功能

##### 1.1 初始化功能 (`initialize()`)

- 初始化消息计数器 `counter = 0`
- 从参数读取最大消息数 `limit`
- **tic 节点特殊行为：** 在初始化时主动发送第一条消息
- 输出初始化日志信息

##### 1.2 消息处理功能 (`handleMessage(cMessage *msg)`)

- **计数器递增：** 每次收到消息，计数器 +1
- **消息名称切换：** 动态修改消息名称
  - tic 收到消息 → 改名为 toc → 转发
  - toc 收到消息 → 改名为 tic → 转发
- **停止条件判断：** 当计数器达到 `limit` 时
  - 删除消息
  - 不再转发
  - 模块静默等待仿真结束
- 记录消息处理日志

##### 1.3 统计输出功能 (`finish()`)

- 输出模块名称和累计接收消息数
- 使用 `recordScalar()` 记录标量统计数据

#### 工作流程图

```
初始化阶段:
┌─────────────┐
│   初始化    │
└──────┬──────┘
       │
       ▼
┌─────────────┐     是     ┌─────────────┐
│  是tic节点? │───────────→│ 发送第一条   │
└──────┬──────┘            │   消息      │
       │否                 └─────────────┘
       ▼
┌─────────────┐
│   等待消息  │
└─────────────┘

消息处理阶段:
┌─────────────┐
│  收到消息   │
└──────┬──────┘
       │
       ▼
┌─────────────┐     是     ┌─────────────┐
│ counter+1   │───────────→│  删除消息   │
└──────┬──────┘            │  结束处理   │
       │否                 └─────────────┘
       ▼
┌─────────────┐
│ 切换消息名  │
│ (tic↔toc)   │
└──────┬──────┘
       │
       ▼
┌─────────────┐
│  转发消息   │
└─────────────┘
```

### 2. TicTocNet 网络（`src/TicTocNet.ned`）

TicTocNet 是网络的顶层定义，采用 **复合模块（Compound Module）** 结构。

#### 网络结构

```
┌─────────────────────────────────────────────┐
│              TicTocNet                       │
│                                             │
│  ┌─────────┐              ┌─────────┐      │
│  │   tic   │←─────────────→│   toc   │      │
│  │  (Txc)  │    双向连接    │  (Txc)  │      │
│  └─────────┘              └─────────┘      │
│                                             │
└─────────────────────────────────────────────┘
```

#### 子模块配置

| 子模块名 | 类型 | 图形显示 | 说明 |
|----------|------|----------|------|
| `tic` | Txc | 路由块图标 | 发送方/接收方节点1 |
| `toc` | Txc | 路由块图标 | 发送方/接收方节点2 |

#### 连接配置

| 连接 | 起点 | 终点 | 说明 |
|------|------|------|------|
| `tic.out` → `toc.in` | tic节点输出 | toc节点输入 | tic向toc发送消息 |
| `toc.out` → `tic.in` | toc节点输出 | tic节点输入 | toc向tic发送消息 |

## 配置参数说明

配置文件位置：`simulations/omnetpp.ini`

```ini
[General]
network = TicTocNet              # 网络名称
description = "TicToc 简单示例"  # 仿真描述
sim-time-limit = 1000s           # 仿真时间限制

# Cmdenv输出配置
**.cmdenv-log-level = INFO       # 日志级别：INFO 显示详细信息
```

### 参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| `network` | TicTocNet | 指定要仿真的网络 |
| `description` | "TicToc 简单示例" | 仿真配置描述 |
| `sim-time-limit` | 1000s | 仿真最多运行1000秒 |
| `**.cmdenv-log-level` | INFO | 命令行环境日志级别 |

## 运行方式

### 编译

```bash
cd /home/sfm/文档/omnet-project-1/tictoc
make -j4
```

### 命令行模式（Cmdenv）

```bash
source /home/sfm/omnetpp-6.2.0/setenv
./out/clang-release/TicToc -u Cmdenv -f simulations/omnetpp.ini -n src:simulations
```

### 图形模式（Qtenv）

```bash
source /home/sfm/omnetpp-6.2.0/setenv
./out/clang-release/TicToc -u Qtenv -f simulations/omnetpp.ini -n src:simulations
```

> **注意：** 在图形模式下，需要手动点击"运行"按钮启动仿真

### 参数说明

| 参数 | 说明 |
|------|------|
| `-u Cmdenv/Qtenv` | 选择用户界面（Cmdenv=命令行，Qtenv=图形） |
| `-f <file>` | 指定配置文件路径 |
| `-n <paths>` | NED 文件搜索路径 |

## 扩展方向

当前 TicToc 示例仅演示了 OMNeT++ 的基础功能。如需构建更复杂的仿真系统，可考虑以下扩展：

1. **多节点网络：** 增加更多通信节点
2. **分层模块：** 使用复合模块构建分层网络架构
3. **自定义消息：** 使用 `.msg` 文件定义结构化消息
4. **统计收集：** 使用 Signals 系统收集详细统计数据
5. **随机变量：** 引入随机延迟、丢包率等网络特性
6. **协议栈集成：** 集成 INET 框架实现标准网络协议
